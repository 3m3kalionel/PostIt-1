    Introduction

    Welcome to the PostIt API!

    PostIt is a simple application that implements a distributed messaging system. It allows friends and colleagues create groups for messages and notifications. You can use our API to access PostIt API endpoints, which serves different functions which includes creating groups, adding user to groups, posting messages to groups and much more.

    Authentication

    PostIt uses email and password authentication to allow access to the API. You can register through the signup endpoint and login through the signin endpoint. Each returns a json web token in response

    PostIt expects for the JSON web token to be included in all subsequent API requests to the server

    Tokens can be embedded in the following ways:

    • request body: As part of the data sent in the request body via key token
    {
  ...
  "token": "abcdef123456"
}
    • request header: As part of the metadata sent in request headers via key Authorization
    Authorization: "Bearer abcdef123456"
    • request query: As part of the query string in request url
    https://app-postit.herokuapp.com/?token=abcdef123456
    • request cookie: It can be set as a cookie on the client so that it is sent with every request to the api

    Sign Up User

    This endpoint creates a user account

    Request Payload:

    {
  "username"
  "email"
  "phoneNo" 
  "password"
}

    Response

    {
  "user": {
    "username"
    "email"
    "phoneNo"
    "status"
    "photoURL"
  },
  "token"
  "message": "Authentication Successful"
}

    HTTP Request

    POST https://app-postit.herokuapp.com/api/v0/user/signup

    Request Payload

    Parameter required/optional Description
    username required a string which represents the name of the user in the application
    password required a string which represents the user password
    email required a string which represents the email address of the user
    phoneNo required a string which respresents the mobile number of the user

    Sign In User

    This endpoint signs in a user

    Request Payload:

    {
  "username"
  "password"
}

    Response

    {
  "user": {
    "username"
    "email"
    "phoneNo"
    "status"
    "photoURL"
  },
  "token"
  "message": "Authentication Successful"
}

    HTTP Request

    POST https://app-postit.herokuapp.com/api/v0/user/signin

    Request Payload

    Parameter required/optional Description
    username required a string which represents the name of the user in the application
    password required a string which represents the user password

    Send password reset mail

    This endpoint sends a password reset link to a user's email address

    Request Payload:

    {
  "recipient": "email"
}

    Response

    {
  "message"
}

    HTTP Request

    POST https://app-postit.herokuapp.com/api/v0/password/mail

    Request Payload

    Parameter required/optional Description
    recipient required email address of user
    This endpoint does not require auth token

    Reset password

    This endpoint updates user account with a new password

    Request Payload:

    {
  "password"
}

    Response

    {
  "user": {
    "username"
    "phoneNo"
    "email"
    "status"
    "photoURL"
  },
  "message": "User account updated"
}

    HTTP Request

    POST https://app-postit.herokuapp.com/api/v0/user/password/reset

    Request Payload

    Parameter required/optional Description
    password required new password provided by user
    This endpoint requires the embedded token in the action url sent to user's email

    Validate user token

    This endpoint validates user token

    It takes no payload. The token should be sent with the request in one of the ways listed above

    Group

    Create a group

    This endpoint creates a group.

    Request Payload:

    {
  "name"
  "description"
}

    description can be null or empty, this is also valid

    {
  "name"
}

    Response

    {
  "message": "Group created"
}

    HTTP Request

    POST https://app-postit.herokuapp.com/api/v0/group

    Request Payload

    Parameter required/optional Description
    name required the title of the group
    description optional a short description of the group

    Add User to Group

    This endpoint takes username of a user and add user to group specified by groupId. groupId is part of the response returned when a group is created

    Request Payload:

    {
  "username"
}

    Response

    {
  "user": {
    "username"
    "phoneNo"
    "email"
    "status"
    "photoURL"
  },
  "message": "User `username` added to group"
}

    HTTP Request

    POST https://app-postit.herokuapp.com/api/v0/group/:groupId/user

    URL Parameters

    Parameter Description
    groupId The ID of the group to which a user is to be added

    Request payload

    Parameter Description
    username The username of the user to be added

    Post message to Group

    This endpoint posts a message to a group

    Request Payload:

    {
  "text"
  "priority"
}

    Response

    {
  "createdMessage": {
    "id"
    "text"
    "priority"
    "createdAt"
    "updatedAt"
    "AuthorUsername"
    "GroupId"
  },
  "message": "Message posted to group"
}

    HTTP Request

    POST https://app-postit.herokuapp.com/api/v0/group/:groupId/message

    URL Parameters

    Parameter Description
    groupId The ID of the group to which a message is to be added

    Request payload

    Parameter required/optional Description
    text required message
    priority optional normal, urgent or critical
    Priority defaults to normal

    Read group messages

    This endpoint reads one or more messages posted to a group

    Request Payload:

    {
  "messages": ["message1", "message2"]
}

    Response

    {
  "message": "Hi `username`, you read one or more messages"
}

    HTTP Request

    POST https://app-postit.herokuapp.com/api/v0/group/:groupId/messages/read

    URL Parameters

    Parameter Description
    groupId The ID of the group

    Request payload

    Parameter required/optional Description
    messages required an array of message ids

    Get users that read message

    This endpoint gets all users that have read a particular message

    Response: Array of objects

    [
  {
    "createdAt"
    "updatedAt"
    "MessageId"
    "UserUsername"
    "GroupId"
    "read"
  }
]

    HTTP Request

    GET https://app-postit.herokuapp.com/api/v0/group/:groupId/message/:messageId/users

    URL Parameters

    Parameter Description
    groupId The ID of the group to which the message belongs
    messageId The ID of the message

    This endpoint gets all the unread messages of a user. The messages are organized according to groups.

}

    GET https://app-postit.herokuapp.com/api/v0/messages/unread

    Errors

    The PostIt API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- You sent invalid url parameters or request payload
    401 Unauthorized -- You are not authenticated or authentication fails
    403 Forbidden -- You are not a member of the group you are trying to access or you tried to perform an operation that can only be done by the creator of the group
    404 Not Found -- The specified User or Group could not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
